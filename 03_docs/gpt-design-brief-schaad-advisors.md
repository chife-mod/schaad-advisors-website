# Design brief — Schaad Advisors

## 1. Контекст проекта

**Проект:** сайт Schaad Advisors / Dr. Jakob Schaad  
**Клиент:** Jakob Schaad  
**Посредник / stakeholder:** Verena  
**Формат:** премиальный персональный advisory-сайт с возможностью дальнейшей интеграции в WordPress.  
**Исходные материалы:** HTML-wireframe, логотип SVG, визуалы бизнес-карточки, фирменная цветовая система, mockup business cards.

Сайт должен транслировать образ умного, спокойного, вдумчивого эксперта на стыке экономики, политики, финансов, регулирования и стратегического advisory. Ключевой принцип: **«так просто, как возможно, но не проще»**.

## 2. Образ бренда

Schaad Advisors — это не агентство и не громкая консалтинговая витрина. Это персональная экспертная практика с высоким уровнем доверия, интеллектуальной глубины и дискретности.

Ключевые качества бренда:

- умный;
- сдержанный;
- стратегический;
- швейцарски точный;
- премиальный без показной роскоши;
- взрослый, авторитетный, но не тяжелый;
- простой, но не примитивный;
- современный, но без визуального шума.

## 3. Целевая аудитория

Основная аудитория — зрелые профессионалы, примерно 55–70 лет:

- топ-менеджеры;
- владельцы бизнеса;
- представители фондов и ассоциаций;
- аналитики;
- ученые;
- представители regulated industries;
- люди, работающие с политикой, финансами, энергетикой, сырьевыми рынками, международными институтами.

Для этой аудитории особенно важны:

- крупная, комфортная типографика;
- ясная навигация;
- высокая контрастность;
- отсутствие перегруженных эффектов;
- быстрый доступ к сути;
- ощущение надежности и интеллектуального контроля.

## 4. Коммуникационное позиционирование

Основная смысловая рамка:

> Expertise at the intersection of economy, politics, regulation and strategic advisory.

Текущий claim из материалов:

> Wissen bündeln. Interessen wahren. Brücken bauen.

Эта фраза хорошо передает характер бренда и может быть сохранена как один из центральных messaging-элементов.

## 5. Структура сайта по текущему wireframe

Текущий HTML-прототип содержит следующие основные разделы:

### Startseite

- Hero с именем Dr. Jakob Schaad;
- claim: Wissen bündeln. Interessen wahren. Brücken bauen.;
- короткое описание экспертности;
- CTA на контакт;
- блок услуг;
- CTA-секция;
- FAQ;
- LinkedIn-блок;
- footer.

### Über mich

- экспертное позиционирование;
- биография и карьерные станции;
- блок «Der Perspektivenwechsel als strategischer Mehrwert»;
- текущие engagements;
- партнерская сеть;
- CTA на контакт.

### Kontakt

- короткий intro-блок;
- форма обращения;
- контактные параметры;
- LinkedIn.

### Lesezeichen

- подборка книг, статей и интеллектуальных ориентиров;
- фильтры по категориям;
- карточки с take-away.

## 6. Визуальное направление

### Общая идея

Сайт должен развивать уже найденный язык бизнес-карточки: крупная типографика, глубокий сине-графитовый фон, тонкие вертикальные линии, много воздуха, строгая сетка, интеллектуальная асимметрия.

Визуальный принцип: **smart Swiss minimalism with a slight edge**.

Не делать:

- тяжелый corporate stock look;
- банальные фото офисов, рукопожатий, небоскребов;
- чрезмерные анимации;
- tech-startup эстетику;
- глянцевую luxury-стилистику;
- слишком мелкую типографику.

Делать:

- спокойные композиции;
- крупные акценты;
- тонкие линии;
- уверенные отступы;
- редкие, точные микроанимации;
- ощущение editorial / institutional / premium advisory.

## 7. Цветовая палитра

Базироваться на существующем фирменном стиле:

- глубокий сине-графитовый / charcoal navy;
- основной синий Schaad Advisors;
- светлый голубовато-серый;
- белый;
- мягкий светло-серый фон;
- опционально — очень дозированный теплый акцент, ближе к muted gold, только для микродеталей.

Важно: не расширять палитру без необходимости. Премиальность здесь строится не на цветах, а на пропорциях, типографике и тишине интерфейса.

## 8. Типографика

Текущий прототип использует контрастную пару:

- serif для заголовков;
- sans-serif для интерфейса и основного текста.

Направление верное. Нужно только уточнить финальную пару шрифтов с учетом производительности и лицензирования.

Рекомендация:

- заголовки: элегантный serif с высоким уровнем доверия;
- основной текст: нейтральный, хорошо читаемый grotesk / sans;
- размер body text не мельчить;
- межстрочный интервал держать комфортным;
- CTA и навигация — uppercase, но аккуратно и не слишком мелко.

## 9. UX-принципы

- Навигация должна быть очевидной и короткой.
- Первый экран должен сразу объяснять: кто он, в чем сила, чем полезен.
- CTA не должен быть агрессивным.
- Контакт должен ощущаться как личное обращение, а не лидогенерационная форма.
- Страницы должны читаться как хорошо структурированный advisory memo.
- FAQ полезен для снятия барьеров и должен остаться.
- Lesezeichen может стать сильным differentiator: показывает интеллект, вкус и профессиональную оптику.

## 10. Анимация и интерактив

Анимация допустима только как тонкая поддержка:

- мягкое появление блоков при scroll;
- деликатные hover-состояния;
- аккордеоны без резких движений;
- возможно, тонкая работа с вертикальными линиями как фирменным мотивом.

Не использовать:

- parallax как основной эффект;
- сложные canvas-анимации;
- тяжелые transition-сцены;
- декоративные эффекты без смысла.

Приоритет: скорость, чистота, доступность.

## 11. Технические требования

Сайт планируется для интеграции в WordPress. Поэтому фронтенд должен быть подготовлен как чистая, переносимая верстка.

Требования:

- чистый HTML;
- CSS отдельно;
- JS отдельно;
- без тяжелых фреймворков на клиенте;
- mobile-first или аккуратная адаптивная сетка;
- понятная компонентная структура;
- классы с предсказуемым naming convention;
- изображения и SVG в assets;
- минимум внешних зависимостей;
- отсутствие inline-стилей в финальной версии;
- готовность к разрезке на WordPress template parts.

Рекомендуемый стек для верстки:

- HTML5;
- CSS custom properties;
- modern CSS grid / flex;
- vanilla JS для меню, аккордеонов и фильтров;
- без jQuery, если WordPress-тема не требует обратного;
- оптимизированные SVG / WebP / AVIF;
- progressive enhancement.

## 12. WordPress-интеграция

Предусмотреть будущую структуру:

- `header.php`;
- `footer.php`;
- `front-page.php`;
- `page-about.php`;
- `page-contact.php`;
- `page-bookmarks.php`;
- reusable template parts для cards, FAQ, CTA, service blocks;
- ACF-поля или native blocks для редактируемого контента;
- отдельные CSS/JS bundles.

Контент должен быть не «зашит» в верстку намертво, а подготовлен к переносу в редактируемые поля.

## 13. Accessibility

Особенно важно из-за возрастной аудитории:

- достаточный размер шрифта;
- контраст минимум WCAG AA;
- понятные focus states;
- клавиатурная навигация;
- aria-expanded для аккордеонов;
- корректные label для формы;
- не полагаться только на цвет для передачи смысла;
- не использовать слишком тонкие шрифты для основного текста.

## 14. Производительность

Цель — быстрый, легкий сайт:

- минимум JS;
- оптимизация шрифтов;
- preload ключевого шрифта после утверждения;
- lazy loading для изображений;
- SVG-логотип без лишнего веса;
- критический CSS можно рассмотреть на этапе production;
- Lighthouse как ориентир: Performance / Accessibility / Best Practices / SEO — 90+.

## 15. Контентные замечания

Текущий текстовый контент выглядит уже достаточно сильным и зрелым. Важно сохранить тон:

- экспертный;
- спокойный;
- не рекламный;
- без маркетинговой гиперболы;
- с уважением к интеллекту читателя.

Нужно дополнительно уточнить:

- финальная биография Jakob Schaad;
- реальные фото;
- точный юридический footer;
- LinkedIn URL;
- privacy / imprint тексты;
- форма: куда отправлять запросы;
- нужны ли мультиязычные версии.

## 16. Первые дизайн-задачи

1. Зафиксировать визуальную систему: цвет, типографика, сетка, отступы.
2. Переработать HTML-wireframe в более чистую дизайн-структуру.
3. Создать homepage direction: hero, services, CTA, FAQ, LinkedIn.
4. Проверить адаптивность на 1440 / 1200 / 1024 / 768 / 390.
5. После утверждения главной — масштабировать систему на остальные страницы.
6. Отдельно подготовить production-ready frontend для WordPress-интеграции.

## 17. Критерии успеха

Сайт успешен, если он ощущается как:

- интеллектуальный;
- премиальный;
- швейцарски точный;
- спокойный;
- быстрый;
- персональный;
- убедительный для зрелой профессиональной аудитории;
- легко интегрируемый в WordPress.

Финальная эмоция: **«здесь человек, которому можно доверить сложный вопрос»**.
